<task type="auto">
  <name>Task 1: Admin channel queries, server actions, and tests</name>
  <files>src/lib/db/queries/admin.ts, src/lib/db/queries/index.ts, src/lib/actions/channels.ts, src/__tests__/admin-channels.test.ts</files>
  <action>
1. Create `src/lib/db/queries/admin.ts`:
   - `getAdminChannels()`: Return all channels with creator name, platform name, topic count, tag count, isEditorsPick. Order by updatedAt desc. Used for the admin list page.
   - `getAdminChannelById(id: string)`: Return a channel with ALL relations loaded (creator, platform, channelTopics with topic+category, channelTags with tag, contentHighlights, relatedFrom with relatedChannel). Used for the edit form.
   - `getAdminFormData()`: Return all creators, platforms, categories (with topics), and tags. Used to populate select/multi-select options on the channel form.

2. Update `src/lib/db/queries/index.ts` to add `export * from './admin'`.

3. Create `src/lib/actions/channels.ts`:
   - `'use server'` directive.
   - `createChannel(formData: FormData)`: Parse form data including name, slug (auto-generate from name if empty), creatorId (or create new creator inline), platformId, platformUrl, description, editorialSummary, isEditorsPick. Insert channel, then insert junction rows for channelTopics, channelCategories, channelTags, contentHighlights. Accept topics/tags/highlights as JSON strings in formData. Revalidate `/admin/channels`, `/`, and relevant category/topic paths. Redirect to /admin/channels.
   - `updateChannel(id: string, formData: FormData)`: Similar to create but uses update + delete-and-reinsert pattern for junction tables. Update updatedAt timestamp.
   - `deleteChannel(id: string)`: Delete channel (cascade handles junctions). Revalidate paths. Redirect to /admin/channels.
   - `toggleEditorsPick(id: string, value: boolean)`: Quick toggle for the list page. Update isEditorsPick, revalidate.

4. Create `src/__tests__/admin-channels.test.ts`:
   - Test slug generation from channel name (slugify logic).
   - Test form data parsing extracts expected fields.
   - Test that JSON string fields (topics, tags, highlights) parse correctly.
   - Mock DB calls -- test the action logic, not the database.
  </action>

  </verify>
  <done>Admin queries return channels with relations, CRUD server actions handle create/update/delete with junction table management, tests pass for form parsing and slug generation.</done>
</task>

2. Create `src/components/admin/channel-form.tsx`:
   - Client component (`'use client'`).
   - Props: `channel?` (for edit mode), `formData` (creators, platforms, categories with topics, tags from getAdminFormData).
   - Single form that handles both create and edit modes.
   - Fields: name, slug (auto-fill from name with edit override), creator (select from existing or create new -- input with datalist), platform (select), platformUrl, description (textarea), editorialSummary (MarkdownField), isEditorsPick (checkbox).
   - **Inline related entities (per user decision -- one form for everything):**
     - Topics: multi-select grouped by category. Use checkboxes organized under category headings.
     - Tags: multi-select checkboxes from all tags.
     - Content highlights: dynamic list with add/remove. Each has title, url, description, displayOrder. Stored as JSON in hidden input.
     - Related channels: searchable select to pick from existing channels. Dynamic list with add/remove and displayOrder.
   - Submit button calls createChannel or updateChannel server action.
   - Delete button (edit mode only) with `window.confirm()` before calling deleteChannel.
   - Use shadcn/ui components: Input, Label, Textarea, Button, Card, Select, Checkbox, Badge.
