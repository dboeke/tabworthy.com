  <behavior>
    - searchChannels("cooking") returns channels ranked by relevance (name matches rank highest)
    - searchChannels("star trek podcasts") with no results falls back to taxonomy broadening
    - searchTaxonomy("science") returns matching categories and topics
    - autocomplete("cook") returns fuzzy name matches from channels, categories, topics, tags
    - getNewestChannels(10) returns 10 channels ordered by created_at DESC
    - getEditorsPicks(10) returns channels where isEditorsPick = true
    - Search API returns empty results for queries under 2 characters
    - Search API accepts optional category constraint parameter
  </behavior>
  <action>
1. Create `src/lib/db/queries/search.ts`:
   - `searchChannels(query: string, options?: { categorySlug?: string, limit?: number })`:
     Use `websearch_to_tsquery('english', query)` against weighted tsvector (name=A, description=B, editorialSummary=C). Order by `ts_rank`. Include channel with creator, platform, tags. Default limit 20. If categorySlug provided, join through channelCategories or channelTopics to filter by category.
   - `searchTaxonomy(query: string)`:
     Search categories and topics by name using `pg_trgm` similarity + ILIKE. Return top 5 categories and top 10 topics (with their category), ordered by similarity.
   - `broadenSearch(query: string)`:
     When channel search returns no results, search taxonomy for matching topics/categories, then return channels from the best-matching topic/category. Return results with a `broadened: true` flag and the matched taxonomy entity name.
   - `autocomplete(query: string)`:
     Use `pg_trgm` similarity + ILIKE on names across channels, categories, topics, tags. Return top 5 from each, ordered by similarity. For fast response.
   - `getNewestChannels(limit = 10)`:
     Select channels ordered by `createdAt DESC`, limit N. Include creator, platform, tags.
   - `getEditorsPicks(limit = 10)`:
     Select channels where `isEditorsPick = true`, ordered by `updatedAt DESC`. Include creator, platform, tags.

2. Update `src/lib/db/queries/index.ts` to add `export * from './search'`.

3. Create `src/app/api/search/route.ts`:
   - GET endpoint. Parse `q` (query), `category` (optional constraint), and `type` (optional: 'channels' | 'taxonomy' | 'all', default 'all') from searchParams.
   - If `q` length < 2, return `{ channels: [], categories: [], topics: [] }`.
   - Call `searchChannels` and `searchTaxonomy` based on type.
   - If channels empty, call `broadenSearch` and include `broadened` flag + message in response.
   - Return JSON response.

4. Create `src/__tests__/search.test.ts`:
   - Test search query construction (websearch_to_tsquery receives user input safely).
   - Test that queries under 2 chars return empty.
   - Test broadenSearch fallback logic.
   - Test autocomplete returns results from multiple entity types.
   - Mock DB layer.

5. Create `src/__tests__/feeds.test.ts`:
   - Test getNewestChannels returns channels sorted by createdAt DESC.
   - Test getEditorsPicks filters by isEditorsPick = true.
   - Mock DB layer.
