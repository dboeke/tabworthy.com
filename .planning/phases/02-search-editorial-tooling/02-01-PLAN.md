  <name>Task 1: Schema migration and admin auth infrastructure</name>
  <files>src/lib/db/schema.ts, src/lib/auth/session.ts, src/lib/actions/auth.ts, src/proxy.ts, src/__tests__/auth.test.ts, src/__tests__/setup.ts</files>
  <action>
1. Install dependencies: `npm install iron-session @dnd-kit/core @dnd-kit/sortable @dnd-kit/utilities @uiw/react-md-editor react-markdown remark-gfm` (install all Phase 2 deps at once to avoid repeated installs).

2. Update `src/lib/db/schema.ts`:
   - Add `isEditorsPick: boolean('is_editors_pick').notNull().default(false)` to the channels table.
   - Add GIN index for full-text search on channels using `index('channels_search_idx').using('gin', sql\`...\`)` with weighted tsvector (name=A, description=B, editorialSummary=C).
   - Add GIN index for pg_trgm on channel names: `index('channels_name_trgm_idx').using('gin', sql\`${channels.name} gin_trgm_ops\`)`.
   - Add similar trgm indexes on categories.name, topics.name, tags.name for autocomplete.

3. Run `npx drizzle-kit push` to apply schema changes.

4. Enable pg_trgm extension on the database. Create a SQL migration file `drizzle/0001_enable_pg_trgm.sql` with `CREATE EXTENSION IF NOT EXISTS pg_trgm;` and execute it via `npx drizzle-kit push` or direct SQL execution.

5. Create `src/lib/auth/session.ts`:
   - Export `SessionData` interface with `isLoggedIn: boolean`.
   - Export `sessionOptions` using iron-session config: password from `process.env.ADMIN_SESSION_SECRET!`, cookieName `admin_session`, httpOnly, sameSite lax, secure in production, maxAge 30 days.
   - Export `async function getSession()` that uses `cookies()` from next/headers and returns `getIronSession<SessionData>`.

6. Create `src/lib/actions/auth.ts`:
   - `'use server'` directive.
   - Export `login(formData: FormData)`: compare password field against `process.env.ADMIN_PASSWORD`, set session.isLoggedIn = true and save on match, return `{ error: 'Invalid password' }` on mismatch. Redirect to `/admin/channels` on success.
   - Export `logout()`: destroy session, redirect to `/admin/login`.

7. Create `src/proxy.ts`:
   - Export `function proxy(request: NextRequest)` that checks if route starts with `/admin`, is not `/admin/login`, and has no `admin_session` cookie -- redirects to `/admin/login`. Otherwise NextResponse.next().
   - Export config matcher for `/admin/:path*`.

8. Create `src/__tests__/auth.test.ts`:
   - Test that login action returns error for invalid password (mock process.env).
   - Test session options have correct cookie config (30 day maxAge, httpOnly).
   - Note: These test the pure logic; proxy behavior is integration-tested manually.

9. Update `src/__tests__/setup.ts` to add any needed env variable mocks for ADMIN_PASSWORD and ADMIN_SESSION_SECRET.
  </action>

  </verify>
