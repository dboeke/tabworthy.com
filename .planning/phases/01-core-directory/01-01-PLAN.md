  <action>
1. Create `drizzle.config.ts`:
   ```typescript
   import { defineConfig } from 'drizzle-kit'

   export default defineConfig({
     schema: './src/lib/db/schema.ts',
     out: './drizzle',
     dialect: 'postgresql',
     dbCredentials: {
       url: process.env.DATABASE_URL!,
     },
   })
   ```

2. Create `src/lib/db/schema.ts` with the complete schema using Drizzle ORM syntax.
   Tables (all with UUID primary keys via `uuid().primaryKey().defaultRandom()`):

   - **platforms**: id, name (unique), displayName, iconUrl, baseUrl, createdAt
   - **categories**: id, name, slug (unique), description, displayOrder, createdAt
   - **topics**: id, categoryId (FK->categories), name, slug, description, displayOrder, createdAt. Add unique constraint on (categoryId, slug).
   - **creators**: id, name, slug (unique), bio, createdAt
   - **channels**: id, creatorId (FK->creators), platformId (FK->platforms), name, slug (unique), platformUrl, description, editorialSummary (text, required for CHAN-05), isPrimary (boolean default false), createdAt, updatedAt
   - **channelTopics**: channelId (FK->channels), topicId (FK->topics), composite PK
   - **channelCategories**: channelId (FK->channels), categoryId (FK->categories), composite PK. This supports channels assigned directly to a category (not just through topics).
   - **tags**: id, name (unique), slug (unique), displayName
   - **channelTags**: channelId (FK->channels), tagId (FK->tags), composite PK
   - **contentHighlights**: id, channelId (FK->channels), title, url, description, displayOrder, createdAt
   - **relatedChannels**: channelId (FK->channels), relatedChannelId (FK->channels), displayOrder, composite PK. Add check constraint channelId != relatedChannelId.

   NOTE: No subscriber_count field -- user decision says "No subscriber counts."
   NOTE: Platform table is a database table, not an enum -- satisfies TECH-04 (platform-agnostic).
   NOTE: Creator-channel separation satisfies CHAN-06 (multi-platform grouping).

   Define relations using Drizzle's `relations()` helper for each table.

3. Create `src/lib/db/index.ts`:
   ```typescript
   import { drizzle } from 'drizzle-orm/neon-http'
   import { neon } from '@neondatabase/serverless'
   import * as schema from './schema'

   const sql = neon(process.env.DATABASE_URL!)
   export const db = drizzle(sql, { schema })
   ```

4. Create `src/lib/db/types.ts` that exports inferred types from schema:
   ```typescript
   import type { InferSelectModel, InferInsertModel } from 'drizzle-orm'
   import * as schema from './schema'

   export type Category = InferSelectModel<typeof schema.categories>
   export type Topic = InferSelectModel<typeof schema.topics>
   export type Creator = InferSelectModel<typeof schema.creators>
   export type Channel = InferSelectModel<typeof schema.channels>
   export type Platform = InferSelectModel<typeof schema.platforms>
   export type Tag = InferSelectModel<typeof schema.tags>
   export type ContentHighlight = InferSelectModel<typeof schema.contentHighlights>
   // ... insert types too
   ```
