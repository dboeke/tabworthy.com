  <action>
1. Create `src/components/seo/json-ld.tsx` -- generic JSON-LD renderer:
   - Server component (avoids duplicate hydration issue from RESEARCH pitfall 3)
   - Uses `schema-dts` types for type safety
   - Renders `<script type="application/ld+json">` with `dangerouslySetInnerHTML`
   - Escape `<` as `\u003c` in JSON output for XSS prevention
   - Per Next.js official JSON-LD guide pattern

2. Create `src/components/taxonomy/breadcrumb-nav.tsx`:
   - Server component
   - Props: `items: Array<{ label: string, href?: string }>` -- last item has no href (current page)
   - Renders: Home > Category > Topic (or Home > Channel Name)
   - Uses shadcn Breadcrumb component if available, or build with nav/ol semantic HTML
   - Styling: "Clean, subtle" per v0 spec -- not the hero of any page
   - TAXO-03: Breadcrumb navigation showing position in hierarchy

3. Create `src/components/seo/breadcrumb-ld.tsx`:
   - Server component
   - Props: same breadcrumb items as visual breadcrumb
   - Generates BreadcrumbList JSON-LD using schema-dts types
   - Uses the JsonLd component to render
   - Each item: `{ "@type": "ListItem", position: N, name: "...", item: "https://tabworthy.com/..." }`
   - Use `https://tabworthy.com` as base URL (or read from env)

4. Create `src/components/seo/category-ld.tsx`:
   - Server component
   - Props: category/topic data with channels
   - Generates ItemList JSON-LD for category and topic pages
   - List items reference channel profile URLs

5. Create `src/components/seo/channel-ld.tsx`:
   - Server component
   - Props: channel with creator and platform data
   - Generates Organization (or Person) JSON-LD
   - Include: name, description, url (platformUrl), sameAs (other platform URLs if multi-platform creator)
  </action>
