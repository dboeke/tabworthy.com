1. Create `src/lib/db/queries/channels.ts` with channel-specific data access functions:

   - `getChannelBySlug(slug: string)`: Returns channel with:
     - creator (with all their other channels + platforms for cross-platform section)
     - platform
     - tags
     - contentHighlights (ordered by displayOrder)
     - relatedChannels (with their creator, platform, tags -- for related channel cards)
     - topics (with their parent categories -- for breadcrumb/categorization)
   - `getAllChannelSlugs()`: Returns all channel slugs for generateStaticParams

   Import db from `../index` and schema from `../schema`. The creator's "other channels" query is key for CHAN-06: fetch all channels where creatorId matches but channel.id != current channel.id, including each channel's platform.

2. Update `src/lib/db/queries/index.ts` to add:
   ```typescript
   export * from './channels'
   ```
   NOTE: If Plan 02 has already added `export * from './taxonomy'`, append to it. If not (parallel execution), just add the channels export. The barrel file may be updated by both plans but each adds a different line, and the last writer wins with both lines present since Plan 04 depends on both.

3. Create `src/components/channels/channel-profile.tsx`:
   - Server component that receives the full channel data object
   - Layout sections (vertically stacked, per v0 spec):

   **Header section:**
   - Channel name (h1, large)
   - Creator name (linked to first category the channel belongs to, or just text)
   - Platform badges row (PlatformBadge components -- see Task 2)
   - Tags as Badge pills

   **Editorial section ("Why Watch"):**
   - The editorialSummary rendered as styled prose
   - Conversational, opinionated voice (content from seed data)
   - 2-3 sentences, styled like a magazine write-up
   - This is the CENTERPIECE -- give it visual prominence with larger text, generous spacing

   **Best Of section (conditional):**
   - Only rendered if contentHighlights.length > 0
   - Uses BestOfList component (Task 2)

   **Cross-platform section (conditional):**
   - Only rendered if creator has more than 1 channel
   - Uses CrossPlatformLinks component (Task 2)
   - Shows "Also on..." with the creator's other platform presences

   **Related channels section (conditional):**
   - Only rendered if relatedChannels.length > 0
   - Uses RelatedChannels component (Task 2)
   - "You might also like" heading

   CRITICAL: Empty optional sections are HIDDEN, not shown with placeholders (user decision). Use simple conditional rendering: `{contentHighlights.length > 0 && <BestOfList ... />}`

   No subscriber counts anywhere (user decision).

   - `generateStaticParams` using `getAllChannelSlugs()`
   - `dynamicParams = true` for ISR
   - `revalidate = 3600`
